## What are metrics?

Metrics are exported columns that can be displayed to the user or used in data analysis.

### Properties of metrics

- Metrics are a function of at least one column(s) in the _same_ table.
- Metrics are N-dimensional vectors, where N is the number of rows in the dataset that is currently being analyzed.
- Metrics are scoped per database.

## Configuration

Metrics can be configured per table inside the main global configuration.

"metrics" should be a key under the respective table schema that has a value of an array of metrics to export.

Each metric to export should contain the name of the exported metric, the function to apply before export, the columns to pass into the function, and optionally the arguments to pass into the function.

The function must return a vector with the same length as the number of rows inside the current dataset relating to the current table.

If you do not want a function to be applied, use the identity function to return the same output as the input.

The function should throw an error if the arguments are invalid (e.g. datatype or input vector lengths mismatch).

The vector arguments (the data) should be supplied as an array of strings. This array is validated.

The (positional) arguments should be supplied as a potentially empty array.

The general shape of a single metric is:

```
name: string
function: string
data: Array<string>
args?: Array<Unknown>
```

Here are two examples of metrics:

```
metrics:
  - name: Qualitative Time Efficiency
    function: identity
    data: [Time Efficiency]
```

```
metrics:
  - name: Duration
    function: timeElapsed
    data: [Start Time, End Time]
    args: []
```

Metric name collision is not detected or reported. It might cause undefined behavior.

Datatype checking may or may not be detected or reported. Please consult the documentation for the specific function to find out.

## Recommendations

I recommend you use metrics to:

- \Export individual columns of data.
- Evaluate implied metrics (i.e. distance traveled, time elapsed)
- Prevent repeated calculations.
