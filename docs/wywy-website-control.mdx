## Wywy-Website-Control Repository

This repo contains all of the configuration and documentation for the entire website. It does not include blog content or database data.

## Scripts

| Path relative to /etc/Wywy-Website-Control | Purpose                                                                                                                                              |
| :----------------------------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------- |
| enter.sh                                   | Enter a docker container running on the current machine                                                                                              |
| install.sh                                 | Install website services.                                                                                                                            |
| pull.sh                                    | Pulls the most recent code available for all Wywy-Website related repositories on this machine. Also recursively initializes and updates submodules. |
| purge.sh                                   | Kills and deletes all docker containers on the current machine.                                                                                      |
| run.sh                                     | Run a website service in development mode. Pass in an argument specifying which service to run.                                                      |
| scripts/run/\[service_name\].sh            | Run a specific website service. Pass in an argument specifying if it's production (prod) or development (dev).                                       |
| scripts/install/\[service_name\].sh        | Install a website service on the current machine. Pass in an argument specifying which service to install.                                           |
| gettimestamp.sh                            | Gets the current date. This should be used when writing documentation.                                                                               |
